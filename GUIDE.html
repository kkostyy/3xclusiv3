01 · VERCEL
шаг первый
Деплой Mini App на Vercel
подготовка
Загрузите два HTML на GitHub
Создайте новый репозиторий на GitHub. Загрузите в корень только два файла:
miniapp.html   # клиентская часть — каталог, корзина, заказы
admin.html     # панель владельца — заказы, товары, аналитика
vercel.json    # роутинг (включён в архив)
деплой
Подключите Vercel к репо
Зайдите на vercel.com → New Project → Import Git Repository → выберите ваш репо → Deploy. Никаких настроек не нужно, vercel.json сделает всё сам.
✦
Vercel автоматически выдаст HTTPS-домен вида your-project.vercel.app — он нужен для настройки в BotFather.
urls
Итоговые адреса
https://your-project.vercel.app        # → miniapp.html (для покупателей)
https://your-project.vercel.app/admin  # → admin.html   (для владельца)
02 · BACKEND
шаг второй
Бот + API на Railway
репозиторий
Загрузите весь проект на GitHub
Полный архив (всю папку 3xclusiv33_final/) загрузите в отдельный приватный репо — он будет содержать и бот, и API, и HTML как fallback.
деплой
Подключите Railway
Зайдите на railway.app → New Project → Deploy from GitHub → выберите репо. Railway прочитает Procfile автоматически.
# Procfile (уже в архиве)
web: python main_with_api.py
переменные окружения
Добавьте Environment Variables
В Railway → Variables добавьте:
ПеременнаяЗначениеОткуда
BOT_TOKENтокен бота@BotFather → /token
ADMIN_IDSваш Telegram ID@userinfobot
DATABASE_PATHdata/store.dbоставьте как есть
PORT8080оставьте как есть
url бэкенда
Пропишите API URL в HTML файлах
После деплоя Railway даст URL вида https://xxx.up.railway.app. Найдите в miniapp.html и admin.html строку и замените:
// Найти:
: 'https://your-backend.vercel.app';

// Заменить на:
: 'https://ваш-проект.up.railway.app';
✦
Это нужно сделать ДО финального коммита в Vercel — тогда HTML на Vercel будет знать, куда обращаться за данными.
03 · BOTFATHER
шаг третий
Настройка в BotFather
кнопка меню для покупателей
Menu Button → miniapp.html
1. Откройте @BotFather
2. /mybots → ваш бот → Bot Settings
3. Menu Button → Edit Menu Button URL
4. URL: https://your-project.vercel.app
5. Название: 🛍 Открыть магазин
admin panel
Добавьте команду /admin
В обработчике handlers/common.py добавьте команду для владельца, которая шлёт ссылку на admin-панель с кнопкой Web App:
# handlers/common.py — добавьте команду /admin
from aiogram.types import InlineKeyboardMarkup, InlineKeyboardButton, WebAppInfo

@router.message(Command("admin"))
async def cmd_admin(message: Message):
    if message.from_user.id not in ADMIN_IDS:
        return
    kb = InlineKeyboardMarkup(inline_keyboard=[[
        InlineKeyboardButton(
            text="⚙️ Панель управления",
            web_app=WebAppInfo(url="https://your-project.vercel.app/admin")
        )
    ]])
    await message.answer("Панель владельца", reply_markup=kb)
04 · LOCAL
локальный запуск
Тест на своей машине
# 1. Установите зависимости
pip install -r requirements.txt

# 2. Создайте .env из примера
cp .env.example .env

06 · TROUBLESHOOT
возможные проблемы
Troubleshooting
ПроблемаПричина и решение
Каталог не загружается Проверьте константу API в HTML — она должна указывать на Railway, не на localhost. Проверьте CORS в api.py (allow_origins: *).
Mini App не открывается в Telegram URL должен быть HTTPS (Vercel выдаёт автоматически). Проверьте Menu Button в BotFather — нужен полный URL с https://.
sendData не работает tg.sendData() работает только внутри Telegram. При тесте в браузере — нормальное поведение, заказ уйдёт через /api/checkout.
Бот не стартует на Railway Проверьте BOT_TOKEN в Variables. Посмотрите логи: Railway → Deployments → View Logs.
Railway засыпает На бесплатном плане Railway не засыпает (в отличие от Render). Если используете Render — добавьте UptimeRobot для пинга /health каждые 5 мин.
Admin panel недоступна Убедитесь что vercel.json есть в репо. Путь /admin должен роутиться на admin.html согласно routes в vercel.json.
